---
title: Entitlement
description: Reference for Discord entitlement objects that represent user or guild access to premium offerings in your application.
---

Entitlements in Discord represent that a user or guild has access to a premium offering in your application.

Refer to the [Monetization Overview](/developers/monetization/overview) for more information on how to use entitlements in your app.

## Entitlement object

### Entitlement structure

<ResponseField name="id" type="snowflake" required>
  ID of the entitlement.
</ResponseField>

<ResponseField name="sku_id" type="snowflake" required>
  ID of the SKU.
</ResponseField>

<ResponseField name="application_id" type="snowflake" required>
  ID of the parent application.
</ResponseField>

<ResponseField name="user_id" type="snowflake">
  ID of the user that is granted access to the entitlement's SKU. Optional field.
</ResponseField>

<ResponseField name="type" type="integer" required>
  [Type of entitlement](#entitlement-types).
</ResponseField>

<ResponseField name="deleted" type="boolean" required>
  Whether the entitlement was deleted.
</ResponseField>

<ResponseField name="starts_at" type="?ISO8601 timestamp" required>
  Start date at which the entitlement is valid. `null` for test entitlements.
</ResponseField>

<ResponseField name="ends_at" type="?ISO8601 timestamp" required>
  Date at which the entitlement is no longer valid. `null` for test entitlements.
</ResponseField>

<ResponseField name="guild_id" type="snowflake">
  ID of the guild that is granted access to the entitlement's SKU. Optional field.
</ResponseField>

<ResponseField name="consumed" type="boolean">
  For consumable items, whether the entitlement has been consumed. Optional field.
</ResponseField>

### Entitlement types

| Type                     | Value | Description                                                    |
|--------------------------|-------|----------------------------------------------------------------|
| PURCHASE                 | 1     | Entitlement was purchased by user                              |
| PREMIUM_SUBSCRIPTION     | 2     | Entitlement for Discord Nitro subscription                     |
| DEVELOPER_GIFT           | 3     | Entitlement was gifted by developer                            |
| TEST_MODE_PURCHASE       | 4     | Entitlement was purchased by a dev in application test mode    |
| FREE_PURCHASE            | 5     | Entitlement was granted when the SKU was free                  |
| USER_GIFT                | 6     | Entitlement was gifted by another user                         |
| PREMIUM_PURCHASE         | 7     | Entitlement was claimed by user for free as a Nitro Subscriber |
| APPLICATION_SUBSCRIPTION | 8     | Entitlement was purchased as an app subscription               |

## List entitlements

<Info>
`GET /applications/{application.id}/entitlements`
</Info>

Returns all entitlements for a given app, both active and expired.

### Query string params

<ParamField query="user_id" type="snowflake">
  User ID to look up entitlements for.
</ParamField>

<ParamField query="sku_ids" type="comma-delimited set of snowflakes">
  Optional list of SKU IDs to check entitlements for.
</ParamField>

<ParamField query="before" type="snowflake">
  Retrieve entitlements before this entitlement ID.
</ParamField>

<ParamField query="after" type="snowflake">
  Retrieve entitlements after this entitlement ID.
</ParamField>

<ParamField query="limit" type="integer" default="100">
  Number of entitlements to return (1–100).
</ParamField>

<ParamField query="guild_id" type="snowflake">
  Guild ID to look up entitlements for.
</ParamField>

<ParamField query="exclude_ended" type="boolean" default="false">
  Whether to omit ended entitlements. Defaults to `false` — ended entitlements are included by default.
</ParamField>

<ParamField query="exclude_deleted" type="boolean" default="true">
  Whether to omit deleted entitlements. Defaults to `true` — deleted entitlements are not included by default.
</ParamField>

## Consume an entitlement

<Info>
`POST /applications/{application.id}/entitlements/{entitlement.id}/consume`
</Info>

For One-Time Purchase consumable SKUs, marks a given entitlement for the user as consumed. The entitlement will have `consumed: true` when using [List Entitlements](#list-entitlements).

Returns `204 No Content` on success.

## Create test entitlement

<Info>
`POST /applications/{application.id}/entitlements`
</Info>

Creates a test entitlement to a given SKU for a given guild or user. Discord will act as though that user or guild has entitlement to your premium offering.

This endpoint returns a partial entitlement object. It will **not** contain `subscription_id`, `starts_at`, or `ends_at`, as it's valid in perpetuity.

After creating a test entitlement, reload your Discord client. Your server or user will then have premium access.

### JSON params

<ParamField body="sku_id" type="string" required>
  ID of the SKU to grant the entitlement to.
</ParamField>

<ParamField body="owner_id" type="string" required>
  ID of the guild or user to grant the entitlement to.
</ParamField>

<ParamField body="owner_type" type="integer" required>
  `1` for a guild subscription, `2` for a user subscription.
</ParamField>

### Request example

```json
{
  "sku_id": "999184799365857331",
  "owner_id": "847184799365857999",
  "owner_type": 1
}
```

## Delete test entitlement

<Info>
`DELETE /applications/{application.id}/entitlements/{entitlement.id}`
</Info>

Deletes a currently-active test entitlement. Discord will act as though that user or guild no longer has entitlement to your premium offering.

Returns `204 No Content` on success.
