Messages are the core unit of communication in Discord. They can contain text, embeds, attachments, reactions, and interactive components.

## Message object

<Info>
Fields specific to the `MESSAGE_CREATE` and `MESSAGE_UPDATE` events are listed in the [Gateway documentation](/developers/events/gateway-events#message-create).
</Info>

<Warning>
An app will receive empty values in the `content`, `embeds`, `attachments`, and `components` fields while `poll` will be omitted if they have not configured (or been approved for) the [`MESSAGE_CONTENT` privileged intent (`1 << 15`)](/developers/events/gateway#message-content-intent).
</Warning>

### Message structure

<ResponseField name="id" type="snowflake" required>
  ID of the message
</ResponseField>

<ResponseField name="channel_id" type="snowflake" required>
  ID of the channel the message was sent in
</ResponseField>

<ResponseField name="author" type="user object" required>
  The author of this message. Not guaranteed to be a valid user — if the message is generated by a webhook, the author object corresponds to the webhook's id, username, and avatar.
</ResponseField>

<ResponseField name="content" type="string" required>
  Contents of the message
</ResponseField>

<ResponseField name="timestamp" type="ISO8601 timestamp" required>
  When this message was sent
</ResponseField>

<ResponseField name="edited_timestamp" type="?ISO8601 timestamp" required>
  When this message was edited (or null if never)
</ResponseField>

<ResponseField name="tts" type="boolean" required>
  Whether this was a TTS message
</ResponseField>

<ResponseField name="mention_everyone" type="boolean" required>
  Whether this message mentions everyone
</ResponseField>

<ResponseField name="mentions" type="array of user objects" required>
  Users specifically mentioned in the message
</ResponseField>

<ResponseField name="mention_roles" type="array of role object ids" required>
  Roles specifically mentioned in this message
</ResponseField>

<ResponseField name="mention_channels" type="array of channel mention objects">
  Channels specifically mentioned in this message. Only textual channels visible to everyone in a public guild will ever be included. Only crossposted messages currently include `mention_channels`.
</ResponseField>

<ResponseField name="attachments" type="array of attachment objects" required>
  Any attached files
</ResponseField>

<ResponseField name="embeds" type="array of embed objects" required>
  Any embedded content
</ResponseField>

<ResponseField name="reactions" type="array of reaction objects">
  Reactions to the message
</ResponseField>

<ResponseField name="nonce" type="integer or string">
  Used for validating a message was sent
</ResponseField>

<ResponseField name="pinned" type="boolean" required>
  Whether this message is pinned
</ResponseField>

<ResponseField name="webhook_id" type="snowflake">
  If the message is generated by a webhook, this is the webhook's id
</ResponseField>

<ResponseField name="type" type="integer" required>
  [Type of message](/developers/resources/message#message-types)
</ResponseField>

<ResponseField name="activity" type="message activity object">
  Sent with Rich Presence-related chat embeds
</ResponseField>

<ResponseField name="application" type="partial application object">
  Sent with Rich Presence-related chat embeds
</ResponseField>

<ResponseField name="application_id" type="snowflake">
  If the message is an interaction or application-owned webhook, this is the id of the application
</ResponseField>

<ResponseField name="flags" type="integer">
  [Message flags](/developers/resources/message#message-flags) combined as a bitfield
</ResponseField>

<ResponseField name="message_reference" type="message reference object">
  Data showing the source of a crosspost, channel follow add, pin, or reply message
</ResponseField>

<ResponseField name="message_snapshots" type="array of message snapshot objects">
  The message associated with the `message_reference`. A minimal subset of fields in a message (e.g. `author` is excluded).
</ResponseField>

<ResponseField name="referenced_message" type="?message object">
  The message associated with the message_reference. Only returned for messages with a type of `REPLY` (19), `THREAD_STARTER_MESSAGE` (21), or `CONTEXT_MENU_COMMAND` (23).
</ResponseField>

<ResponseField name="interaction_metadata" type="message interaction metadata object">
  Sent if the message is sent as a result of an interaction
</ResponseField>

<ResponseField name="thread" type="channel object">
  The thread that was started from this message, includes thread member object
</ResponseField>

<ResponseField name="components" type="array of message components">
  Sent if the message contains components like buttons, action rows, or other interactive components
</ResponseField>

<ResponseField name="sticker_items" type="array of message sticker item objects">
  Sent if the message contains stickers
</ResponseField>

<ResponseField name="position" type="integer">
  A generally increasing integer that represents the approximate position of the message in a thread
</ResponseField>

<ResponseField name="role_subscription_data" type="role subscription data object">
  Data of the role subscription purchase or renewal that prompted this `ROLE_SUBSCRIPTION_PURCHASE` message
</ResponseField>

<ResponseField name="poll" type="poll object">
  A poll
</ResponseField>

<ResponseField name="call" type="message call object">
  The call associated with the message
</ResponseField>

## Message flags

| Flag                                     | Value     | Description                                                                             |
|------------------------------------------|-----------|-----------------------------------------------------------------------------------------|
| `CROSSPOSTED`                            | `1 << 0`  | This message has been published to subscribed channels (via Channel Following)          |
| `IS_CROSSPOST`                           | `1 << 1`  | This message originated from a message in another channel (via Channel Following)       |
| `SUPPRESS_EMBEDS`                        | `1 << 2`  | Do not include any embeds when serializing this message                                 |
| `SOURCE_MESSAGE_DELETED`                 | `1 << 3`  | The source message for this crosspost has been deleted (via Channel Following)          |
| `URGENT`                                 | `1 << 4`  | This message came from the urgent message system                                        |
| `HAS_THREAD`                             | `1 << 5`  | This message has an associated thread, with the same id as the message                  |
| `EPHEMERAL`                              | `1 << 6`  | This message is only visible to the user who invoked the Interaction                    |
| `LOADING`                                | `1 << 7`  | This message is an Interaction Response and the bot is "thinking"                       |
| `FAILED_TO_MENTION_SOME_ROLES_IN_THREAD` | `1 << 8`  | This message failed to mention some roles and add their members to the thread           |
| `SUPPRESS_NOTIFICATIONS`                 | `1 << 12` | This message will not trigger push and desktop notifications                            |
| `IS_VOICE_MESSAGE`                       | `1 << 13` | This message is a voice message                                                         |
| `HAS_SNAPSHOT`                           | `1 << 14` | This message has a snapshot (via Message Forwarding)                                    |
| `IS_COMPONENTS_V2`                       | `1 << 15` | Allows you to create fully component-driven messages. Once set, cannot be removed.      |

### Embed limits

| Field          | Limit           |
|----------------|-----------------|
| `title`        | 256 characters  |
| `description`  | 4096 characters |
| `fields`       | Up to 25 field objects |
| `field.name`   | 256 characters  |
| `field.value`  | 1024 characters |
| `footer.text`  | 2048 characters |
| `author.name`  | 256 characters  |

The combined sum of characters in all `title`, `description`, `field.name`, `field.value`, `footer.text`, and `author.name` fields across all embeds must not exceed 6000 characters.

## Allowed mentions object

The `allowed_mentions` field lets you determine whether users will receive notifications when you include mentions in the message content.

### Allowed mention types

| Type                | Value        | Description                           |
|---------------------|--------------|---------------------------------------|
| Role Mentions       | `"roles"`    | Controls role mentions                |
| User Mentions       | `"users"`    | Controls user mentions                |
| Everyone Mentions   | `"everyone"` | Controls @everyone and @here mentions |

### Allowed mentions structure

<ResponseField name="parse" type="array of allowed mention types">
  An array of allowed mention types to parse from the content
</ResponseField>

<ResponseField name="roles" type="array of snowflakes">
  Array of role ids to mention, max 100
</ResponseField>

<ResponseField name="users" type="array of snowflakes">
  Array of user ids to mention, max 100
</ResponseField>

<ResponseField name="replied_user" type="boolean">
  For replies, whether to mention the author of the message being replied to, defaults to false
</ResponseField>

<Info>
The `parse` field is mutually exclusive with the `roles` and `users` fields. You cannot use both `parse: ["users"]` and `users: [...]` in the same request.
</Info>

## Get channel messages

<Note>
**GET** `/channels/{channel.id}/messages`
</Note>

Retrieves the messages in a channel. Returns an array of [message](/developers/resources/message#message-object) objects from newest to oldest on success.

<Info>
The `before`, `after`, and `around` parameters are mutually exclusive, only one may be passed at a time.
</Info>

### Query string params

<ParamField query="around" type="snowflake">
  Get messages around this message ID
</ParamField>

<ParamField query="before" type="snowflake">
  Get messages before this message ID
</ParamField>

<ParamField query="after" type="snowflake">
  Get messages after this message ID
</ParamField>

<ParamField query="limit" type="integer" default="50">
  Max number of messages to return (1-100)
</ParamField>

## Get channel message

<Note>
**GET** `/channels/{channel.id}/messages/{message.id}`
</Note>

Retrieves a specific message in the channel. Returns a [message](/developers/resources/message#message-object) object on success.

## Create message

<Note>
**POST** `/channels/{channel.id}/messages`
</Note>

Post a message to a guild text or DM channel. Returns a [message](/developers/resources/message#message-object) object. Fires a `Message Create` Gateway event.

<Warning>
Discord may strip certain characters from message content, like invalid unicode characters or characters which cause unexpected message formatting. If you are passing user-generated strings into message content, consider sanitizing the data.
</Warning>

### Limitations

- When operating on a guild channel, the current user must have the `SEND_MESSAGES` permission.
- When sending a message with `tts` set to `true`, the current user must have the `SEND_TTS_MESSAGES` permission.
- The maximum request size when sending a message is **25 MiB**.

### JSON/form params

<ParamField body="content" type="string">
  Message contents (up to 2000 characters). At least one of `content`, `embeds`, `sticker_ids`, `components`, `files[n]`, `poll`, or `shared_client_theme` is required.
</ParamField>

<ParamField body="nonce" type="integer or string">
  Can be used to verify a message was sent (up to 25 characters)
</ParamField>

<ParamField body="tts" type="boolean">
  `true` if this is a TTS message
</ParamField>

<ParamField body="embeds" type="array of embed objects">
  Up to 10 `rich` embeds (up to 6000 characters)
</ParamField>

<ParamField body="allowed_mentions" type="allowed mention object">
  Allowed mentions for the message
</ParamField>

<ParamField body="message_reference" type="message reference object">
  Include to make your message a reply or a forward
</ParamField>

<ParamField body="components" type="array of message component objects">
  Components to include with the message
</ParamField>

<ParamField body="sticker_ids" type="array of snowflakes">
  IDs of up to 3 stickers in the server to send in the message
</ParamField>

<ParamField body="attachments" type="array of partial attachment objects">
  Attachment objects with filename and description
</ParamField>

<ParamField body="flags" type="integer">
  Message flags; only `SUPPRESS_EMBEDS`, `SUPPRESS_NOTIFICATIONS`, `IS_VOICE_MESSAGE`, and `IS_COMPONENTS_V2` can be set
</ParamField>

<ParamField body="enforce_nonce" type="boolean">
  If true and nonce is present, it will be checked for uniqueness in the past few minutes
</ParamField>

<ParamField body="poll" type="poll request object">
  A poll
</ParamField>

### Example request body

```json
{
  "content": "Hello, World!",
  "tts": false,
  "embeds": [{
    "title": "Hello, Embed!",
    "description": "This is an embedded message."
  }]
}
```

## Crosspost message

<Note>
**POST** `/channels/{channel.id}/messages/{message.id}/crosspost`
</Note>

Crosspost a message in an Announcement Channel to following channels. Requires `SEND_MESSAGES` permission if the current user sent the message, or `MANAGE_MESSAGES` for all other messages.

## Create reaction

<Note>
**PUT** `/channels/{channel.id}/messages/{message.id}/reactions/{emoji}/@me`
</Note>

Create a reaction for the message. Requires `READ_MESSAGE_HISTORY` permission. Additionally, if nobody else has reacted to the message using this emoji, this endpoint requires `ADD_REACTIONS` permission. Returns a 204 empty response on success.

The `emoji` must be URL Encoded or the request will fail with `10014: Unknown Emoji`. To use custom emoji, encode it in the format `name:id`.

## Delete own reaction

<Note>
**DELETE** `/channels/{channel.id}/messages/{message.id}/reactions/{emoji}/@me`
</Note>

Delete a reaction the current user has made for the message. Returns a 204 empty response on success.

## Delete user reaction

<Note>
**DELETE** `/channels/{channel.id}/messages/{message.id}/reactions/{emoji}/{user.id}`
</Note>

Deletes another user's reaction. Requires the `MANAGE_MESSAGES` permission.

## Search guild messages

<Note>
**GET** `/guilds/{guild.id}/messages/search`
</Note>

Returns a list of messages that match a search query in the guild. Requires the `READ_MESSAGE_HISTORY` permission.

<Warning>
This endpoint is restricted according to whether the `MESSAGE_CONTENT` Privileged Intent is enabled for your application.
</Warning>
