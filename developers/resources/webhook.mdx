---
title: Webhook
description: Reference for Discord webhook objects and management endpoints.
---

Webhooks are a low-effort way to post messages to channels in Discord. They do not require a bot user or authentication to use.

Apps can also subscribe to webhook events (i.e. outgoing webhooks) when events happen _in_ Discord, which is detailed in the [Webhook Events](/developers/events/webhook-events) documentation.

---

## Webhook object

Used to represent a webhook.

### Webhook structure

<ResponseField name="id" type="snowflake" required>
  The id of the webhook.
</ResponseField>

<ResponseField name="type" type="integer" required>
  The [type](#webhook-types) of the webhook.
</ResponseField>

<ResponseField name="guild_id" type="?snowflake">
  The guild id this webhook is for, if any.
</ResponseField>

<ResponseField name="channel_id" type="?snowflake" required>
  The channel id this webhook is for, if any.
</ResponseField>

<ResponseField name="user" type="user object">
  The user this webhook was created by. Not returned when getting a webhook with its token.
</ResponseField>

<ResponseField name="name" type="?string" required>
  The default name of the webhook.
</ResponseField>

<ResponseField name="avatar" type="?string" required>
  The default user avatar [hash](/developers/reference#image-formatting) of the webhook.
</ResponseField>

<ResponseField name="token" type="string">
  The secure token of the webhook. Returned for Incoming Webhooks only.
</ResponseField>

<ResponseField name="application_id" type="?snowflake" required>
  The bot/OAuth2 application that created this webhook.
</ResponseField>

<ResponseField name="source_guild" type="partial guild object">
  The guild of the channel that this webhook is following. Returned for Channel Follower Webhooks. Absent if the webhook creator has since lost access to the guild where the followed channel resides.
</ResponseField>

<ResponseField name="source_channel" type="partial channel object">
  The channel that this webhook is following. Returned for Channel Follower Webhooks. Absent if the webhook creator has since lost access to the guild where the followed channel resides.
</ResponseField>

<ResponseField name="url" type="string">
  The url used for executing the webhook. Returned by the [webhooks](/developers/topics/oauth2#webhooks) OAuth2 flow.
</ResponseField>

### Webhook types

<Info>
These types don't include [webhook events](/developers/events/webhook-events), which are outgoing webhooks sent to your app by Discord. See [Webhook Events](/developers/events/webhook-events) for details.
</Info>

| Value | Name               | Description                                                                                                     |
|-------|--------------------|-----------------------------------------------------------------------------------------------------------------|
| `1`   | Incoming           | Incoming Webhooks can post messages to channels with a generated token                                          |
| `2`   | Channel Follower   | Channel Follower Webhooks are internal webhooks used with Channel Following to post new messages into channels  |
| `3`   | Application        | Application webhooks are webhooks used with Interactions                                                        |

## Create webhook

<Note>
**POST** `/channels/{channel.id}/webhooks`
</Note>

Creates a new webhook and returns a [webhook](#webhook-object) object on success. Requires the `MANAGE_WEBHOOKS` permission. Fires a [Webhooks Update](/developers/events/gateway-events#webhooks-update) Gateway event.

A webhook name is valid if:
- It does not contain the substrings `clyde` or `discord` (case-insensitive)
- It follows the nickname guidelines in the [Usernames and Nicknames](/developers/resources/user#usernames-and-nicknames) documentation, with an exception that webhook names can be up to 80 characters

<Info>
This endpoint supports the `X-Audit-Log-Reason` header.
</Info>

### JSON params

<ParamField body="name" type="string" required>
  Name of the webhook (1–80 characters).
</ParamField>

<ParamField body="avatar" type="?image data">
  Image for the default webhook avatar. See [image data](/developers/reference#image-data).
</ParamField>

## Get channel webhooks

<Note>
**GET** `/channels/{channel.id}/webhooks`
</Note>

Returns a list of channel [webhook](#webhook-object) objects. Requires the `MANAGE_WEBHOOKS` permission.

## Get guild webhooks

<Note>
**GET** `/guilds/{guild.id}/webhooks`
</Note>

Returns a list of guild [webhook](#webhook-object) objects. Requires the `MANAGE_WEBHOOKS` permission.

## Get webhook

<Note>
**GET** `/webhooks/{webhook.id}`
</Note>

Returns the [webhook](#webhook-object) object for the given id.

<Info>
This request requires the `MANAGE_WEBHOOKS` permission unless the application making the request owns the webhook.
</Info>

## Get webhook with token

<Note>
**GET** `/webhooks/{webhook.id}/{webhook.token}`
</Note>

Same as [Get webhook](#get-webhook), except this call does not require authentication and returns no user in the webhook object.

## Modify webhook

<Note>
**PATCH** `/webhooks/{webhook.id}`
</Note>

Modify a webhook. Requires the `MANAGE_WEBHOOKS` permission. Returns the updated [webhook](#webhook-object) object on success. Fires a [Webhooks Update](/developers/events/gateway-events#webhooks-update) Gateway event.

<Note>
All parameters to this endpoint are optional.
</Note>

<Info>
This endpoint supports the `X-Audit-Log-Reason` header.
</Info>

### JSON params

<ParamField body="name" type="string">
  The default name of the webhook.
</ParamField>

<ParamField body="avatar" type="?image data">
  Image for the default webhook avatar. See [image data](/developers/reference#image-data).
</ParamField>

<ParamField body="channel_id" type="snowflake">
  The new channel id this webhook should be moved to.
</ParamField>

## Modify webhook with token

<Note>
**PATCH** `/webhooks/{webhook.id}/{webhook.token}`
</Note>

Same as [Modify webhook](#modify-webhook), except this call does not require authentication, does not accept a `channel_id` parameter in the body, and does not return a user in the webhook object.

## Delete webhook

<Note>
**DELETE** `/webhooks/{webhook.id}`
</Note>

Delete a webhook permanently. Requires the `MANAGE_WEBHOOKS` permission. Returns a `204 No Content` response on success. Fires a [Webhooks Update](/developers/events/gateway-events#webhooks-update) Gateway event.

<Info>
This endpoint supports the `X-Audit-Log-Reason` header.
</Info>

## Delete webhook with token

<Note>
**DELETE** `/webhooks/{webhook.id}/{webhook.token}`
</Note>

Same as [Delete webhook](#delete-webhook), except this call does not require authentication.

## Execute webhook

<Note>
**POST** `/webhooks/{webhook.id}/{webhook.token}`
</Note>

Refer to [Uploading Files](/developers/reference#uploading-files) for details on attachments and `multipart/form-data` requests. Returns a message or `204 No Content` depending on the `wait` query parameter.

<Info>
When sending a message, you must provide a value for **at least one of** `content`, `embeds`, `components`, `file`, or `poll`.
</Info>

<Info>
If the webhook channel is a forum or media channel, you must provide either `thread_id` in the query string params or `thread_name` in the JSON/form params. If `thread_id` is provided, the message will send in that thread. If `thread_name` is provided, a thread with that name will be created in the channel.
</Info>

<Warning>
Discord may strip certain characters from message content, like invalid unicode characters or characters which cause unexpected message formatting. If you are passing user-generated strings into message content, consider sanitizing the data to prevent unexpected behavior and using `allowed_mentions` to prevent unexpected mentions.
</Warning>

### Query string params

<ParamField query="wait" type="boolean" default="false">
  Waits for server confirmation of message send before responding. When `false`, a message that is not saved does not return an error.
</ParamField>

<ParamField query="thread_id" type="snowflake">
  Send a message to the specified thread within a webhook's channel. The thread will automatically be unarchived.
</ParamField>

<ParamField query="with_components" type="boolean" default="false">
  Whether to respect the `components` field of the request. When enabled, allows application-owned webhooks to use all components and non-owned webhooks to use non-interactive components.
</ParamField>

### JSON/form params

<ParamField body="content" type="string">
  The message contents (up to 2000 characters). Required if `file`, `embeds`, and `poll` are not provided.
</ParamField>

<ParamField body="username" type="string">
  Override the default username of the webhook.
</ParamField>

<ParamField body="avatar_url" type="string">
  Override the default avatar of the webhook.
</ParamField>

<ParamField body="tts" type="boolean">
  `true` if this is a TTS message.
</ParamField>

<ParamField body="embeds" type="array of embed objects">
  Embedded `rich` content (up to 10 embeds). Required if `content`, `file`, and `poll` are not provided.
</ParamField>

<ParamField body="allowed_mentions" type="allowed mention object">
  Allowed mentions for the message.
</ParamField>

<ParamField body="components" type="array of message component objects">
  The components to include with the message. Application-owned webhooks can always send components. Non-application-owned webhooks cannot send interactive components unless `with_components` is set.
</ParamField>

<ParamField body="files[n]" type="file contents">
  The contents of the file being sent. Required if `content`, `embeds`, and `poll` are not provided. See [Uploading Files](/developers/reference#uploading-files) for details.
</ParamField>

<ParamField body="payload_json" type="string">
  JSON encoded body of non-file params. Only for `multipart/form-data` requests.
</ParamField>

<ParamField body="attachments" type="array of partial attachment objects">
  Attachment objects with filename and description. See [Uploading Files](/developers/reference#uploading-files) for details.
</ParamField>

<ParamField body="flags" type="integer">
  [Message flags](/developers/resources/message#message-object-message-flags) combined as a bitfield. Only `SUPPRESS_EMBEDS`, `SUPPRESS_NOTIFICATIONS`, and `IS_COMPONENTS_V2` can be set. When `IS_COMPONENTS_V2` is set, the webhook message can only contain `components` — providing `content`, `embeds`, `files[n]`, or `poll` will fail with a `400 BAD REQUEST`.
</ParamField>

<ParamField body="thread_name" type="string">
  Name of thread to create. Requires the webhook channel to be a forum or media channel.
</ParamField>

<ParamField body="applied_tags" type="array of snowflakes">
  Array of tag ids to apply to the thread. Requires the webhook channel to be a forum or media channel.
</ParamField>

<ParamField body="poll" type="poll request object">
  A poll. Required if `content`, `file`, and `embeds` are not provided. See [poll create request object](/developers/resources/poll#poll-create-request-object).
</ParamField>

<Info>
For webhook embed objects, you can set every field except `type` (it will be `rich` regardless), `provider`, `video`, and any `height`, `width`, or `proxy_url` values for images.
</Info>

## Execute GitHub-compatible webhook

<Note>
**POST** `/webhooks/{webhook.id}/{webhook.token}/github`
</Note>

Add a new webhook to your GitHub repo (in the repo's settings) and use this endpoint as the "Payload URL." You can choose what events your Discord channel receives by selecting "Let me select individual events" when configuring the webhook.

The supported GitHub [events](https://docs.github.com/en/webhooks/webhook-events-and-payloads) are: `commit_comment`, `create`, `delete`, `fork`, `issue_comment`, `issues`, `member`, `public`, `pull_request`, `pull_request_review`, `pull_request_review_comment`, `push`, `release`, `watch`, `check_run`, `check_suite`, `discussion`, and `discussion_comment`.

</ParamField>

## Edit webhook message

<Note>
**PATCH** `/webhooks/{webhook.id}/{webhook.token}/messages/{message.id}`
</Note>

Edits a previously-sent webhook message from the same token. Returns a [message](/developers/resources/message#message-object) object on success.

When the `content` field is edited, the `mentions`, `mention_roles`, and `mention_everyone` fields in the message object will be reconstructed from scratch based on the new content. The `allowed_mentions` field of the edit request controls how this happens. If there is no explicit `allowed_mentions` in the edit request, the content will be parsed with default allowances, without regard to whether or not an `allowed_mentions` was present in the original request.

Any provided files will be **appended** to the message. To remove or replace files you will need to supply the `attachments` field, which specifies the files to retain on the message after edit.

<Warning>
Starting with API v10, the `attachments` array must contain all attachments that should be present after edit, including **retained and new** attachments provided in the request body.
</Warning>

<Note>
All parameters to this endpoint are optional and nullable.
</Note>

### Query string params

<ParamField query="thread_id" type="snowflake">
  Id of the thread the message is in.
</ParamField>

<ParamField query="with_components" type="boolean" default="false">
  Whether to respect the `components` field of the request. When enabled, allows application-owned webhooks to use all components and non-owned webhooks to use non-interactive components.
</ParamField>

### JSON/form params

<ParamField body="content" type="string">
  The message contents (up to 2000 characters).
</ParamField>

<ParamField body="embeds" type="array of embed objects">
  Embedded `rich` content (up to 10 embeds).
</ParamField>

<ParamField body="flags" type="integer">
  [Message flags](/developers/resources/message#message-object-message-flags) combined as a bitfield. Only `SUPPRESS_EMBEDS` and `IS_COMPONENTS_V2` can be set. `SUPPRESS_EMBEDS` can be both set and unset, while `IS_COMPONENTS_V2` can only be set. When `IS_COMPONENTS_V2` is set, any `content`, `embeds`, `files[n]`, or `poll` values in the initial message must be set to `null` first, otherwise it will fail with a `400 BAD REQUEST`.
</ParamField>

<ParamField body="allowed_mentions" type="allowed mention object">
  Allowed mentions for the message.
</ParamField>

<ParamField body="components" type="array of message component objects">
  The components to include with the message. Application-owned webhooks can always send components. Non-application-owned webhooks cannot send interactive components unless `with_components` is set.
</ParamField>

<ParamField body="files[n]" type="file contents">
  The contents of the file being sent/edited. See [Uploading Files](/developers/reference#uploading-files) for details.
</ParamField>

<ParamField body="payload_json" type="string">
  JSON encoded body of non-file params. Only for `multipart/form-data` requests.
</ParamField>

<ParamField body="attachments" type="array of partial attachment objects">
  Attached files to keep and possible descriptions for new files. See [Uploading Files](/developers/reference#uploading-files) for details.
</ParamField>

<ParamField body="poll" type="poll request object">
  A poll. Polls can only be added when editing a deferred interaction response.
</ParamField>

**DELETE** `/webhooks/{webhook.id}/{webhook.token}/messages/{message.id}`
</Note>

Deletes a message that was created by the webhook. Returns a `204 No Content` response on success.

### Query string params

<ParamField query="thread_id" type="snowflake">
  Id of the thread the message is in.
</ParamField>
