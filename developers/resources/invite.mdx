## Invite object

Represents a code that, when used, adds a user to a guild or group DM channel.

### Invite structure

<ResponseField name="type" type="integer" required>
  The [type of invite](#invite-types).
</ResponseField>

<ResponseField name="code" type="string" required>
  The invite code (unique ID).
</ResponseField>

<ResponseField name="guild" type="partial guild object">
  The guild this invite is for. Optional field.
</ResponseField>

<ResponseField name="channel" type="?partial channel object" required>
  The channel this invite is for. Can be `null`.
</ResponseField>

<ResponseField name="inviter" type="user object">
  The user who created the invite. Optional field.
</ResponseField>

<ResponseField name="target_type" type="integer">
  The [type of target](#invite-target-types) for this voice channel invite. Optional field.
</ResponseField>

<ResponseField name="target_user" type="user object">
  The user whose stream to display for this voice channel stream invite. Optional field.
</ResponseField>

<ResponseField name="target_application" type="partial application object">
  The embedded application to open for this voice channel embedded application invite. Optional field.
</ResponseField>

<ResponseField name="approximate_presence_count" type="integer">
  Approximate count of online members. Returned from `GET /invites/{code}` when `with_counts` is `true`. Optional field.
</ResponseField>

<ResponseField name="approximate_member_count" type="integer">
  Approximate count of total members. Returned from `GET /invites/{code}` when `with_counts` is `true`. Optional field.
</ResponseField>

<ResponseField name="expires_at" type="?ISO8601 timestamp" required>
  The expiration date of this invite. `null` if the invite does not expire.
</ResponseField>

<ResponseField name="guild_scheduled_event" type="guild scheduled event object">
  Guild scheduled event data. Only included if `guild_scheduled_event_id` contains a valid guild scheduled event ID. Optional field.
</ResponseField>

<ResponseField name="flags" type="integer">
  [Guild invite flags](#guild-invite-flags) for guild invites. Optional field.
</ResponseField>

<ResponseField name="roles" type="array of partial role objects">
  The roles assigned to the user upon accepting the invite. Optional field. This is a partial role object containing only `id`, `name`, `position`, `color`, `colors`, `icon`, and `unicode_emoji`.
</ResponseField>

### Invite types

| Type     | Value |
|----------|-------|
| GUILD    | 0     |
| GROUP_DM | 1     |
| FRIEND   | 2     |

### Invite target types

| Type                 | Value |
|----------------------|-------|
| STREAM               | 1     |
| EMBEDDED_APPLICATION | 2     |

### Guild invite flags

| Flag            | Value    | Description                                       |
|-----------------|----------|---------------------------------------------------|
| IS_GUEST_INVITE | `1 << 0` | This invite is a guest invite for a voice channel |

## Invite metadata object

Extra information about an invite. Extends the [invite](#invite-object) object.

### Invite metadata structure

<ResponseField name="uses" type="integer" required>
  Number of times this invite has been used.
</ResponseField>

<ResponseField name="max_uses" type="integer" required>
  Max number of times this invite can be used.
</ResponseField>

<ResponseField name="max_age" type="integer" required>
  Duration in seconds after which the invite expires.
</ResponseField>

<ResponseField name="temporary" type="boolean" required>
  Whether this invite only grants temporary membership.
</ResponseField>

<ResponseField name="created_at" type="ISO8601 timestamp" required>
  When this invite was created.
</ResponseField>

### Invite metadata example

```json
{
  "uses": 0,
  "max_uses": 0,
  "max_age": 0,
  "temporary": false,
  "created_at": "2016-03-31T19:15:39.954000+00:00"
}
```

## Get invite

<Info>
`GET /invites/{invite.code}`
</Info>

Returns an [invite](#invite-object) object for the given code.

### Query string params

<ParamField query="with_counts" type="boolean">
  Whether the invite should contain approximate member counts.
</ParamField>

<ParamField query="guild_scheduled_event_id" type="snowflake">
  The guild scheduled event to include with the invite.
</ParamField>

## Delete invite

<Info>
`DELETE /invites/{invite.code}`
</Info>

Delete an invite. Requires the `MANAGE_CHANNELS` permission on the channel this invite belongs to, or `MANAGE_GUILD` to remove any invite across the guild. Returns an [invite](#invite-object) object on success. Fires an `Invite Delete` gateway event.

<Info>
This endpoint supports the `X-Audit-Log-Reason` header.
</Info>

## Get target users

<Info>
`GET /invites/{invite.code}/target-users`
</Info>

Gets the users allowed to see and accept this invite. Response is a CSV file with the header `user_id` and each user ID on its own line. Requires the caller to be the inviter, or have `MANAGE_GUILD` permission, or have `VIEW_AUDIT_LOG` permission.

## Update target users

<Info>
`PUT /invites/{invite.code}/target-users`
</Info>

Updates the users allowed to see and accept this invite. Uploading a file with invalid user IDs returns a `400` with the invalid IDs described. Requires the caller to be the inviter or have `MANAGE_GUILD` permission.

### Form params

<ParamField body="target_users_file" type="file" required>
  A CSV file with a single column of user IDs for all the users able to accept this invite.
</ParamField>

### Error response example

```json
{
  "target_users_file": [
    "Line 5: invalid user ID - must be a valid Discord snowflake",
    "Line 6: invalid user ID format"
  ]
}
```

## Get target users job status

<Info>
`GET /invites/{invite.code}/target-users/job-status`
</Info>

Processing target users from a CSV when creating or updating an invite is done asynchronously. This endpoint lets you check the status of that job. Requires the caller to be the inviter, or have `MANAGE_GUILD` permission, or have `VIEW_AUDIT_LOG` permission.

### Status codes

| Code | Name        | Description                                                     |
|------|-------------|-----------------------------------------------------------------|
| 0    | UNSPECIFIED | The default value.                                              |
| 1    | PROCESSING  | The job is still being processed.                               |
| 2    | COMPLETED   | The job has been completed successfully.                        |
| 3    | FAILED      | The job has failed. See `error_message` for details.            |

### Response example

```json
{
  "status": 3,
  "total_users": 100,
  "processed_users": 41,
  "created_at": "2025-01-08T12:00:00.000000+00:00",
  "completed_at": null,
  "error_message": "Failed to parse CSV file"
}
```
