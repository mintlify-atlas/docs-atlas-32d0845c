[Applications](/developers/quick-start/overview-of-apps) (or "apps") are containers for developer platform features, and can be installed to Discord servers and/or user accounts.

## Application object

### Application structure

</ResponseField>

<ResponseField name="name" type="string" required>

<ResponseField name="interactions_endpoint_url" type="?string">

</ResponseField>

<ResponseField name="event_webhooks_types" type="array of strings">

<ResponseField name="tags" type="array of strings">

## Application integration types

Where an app can be installed, also called its supported installation contexts.

| Type            | ID | Description                   |
|-----------------|----|-------------------------------|
| `GUILD_INSTALL` | 0  | App is installable to servers |
| `USER_INSTALL`  | 1  | App is installable to users   |

## Application integration type configuration object

<ResponseField name="oauth2_install_params" type="install params object">
  Install params for each installation context's default in-app authorization link
</ResponseField>

## Application event webhook status

Status indicating whether event webhooks are enabled or disabled for an application.

| Name                  | Value | Description                                                       |
|-----------------------|-------|-------------------------------------------------------------------|
| `DISABLED`            | `1`   | Webhook events are disabled by developer                          |
| `ENABLED`             | `2`   | Webhook events are enabled by developer                           |
| `DISABLED_BY_DISCORD` | `3`   | Webhook events are disabled by Discord, usually due to inactivity |

## Installation context

An app's installation context defines how it's installed: to a server, to a user, or both.

The installation context affects how your app can be seen and used within Discord. For example, apps installed only to a user can't take actions in a server, and apps installed only to a server can't be accessed within a user's DMs.

### Server context

Apps installed in a server context (server-installed apps) must be authorized by a server member with the `MANAGE_GUILD` permission. Server-installed apps are visible to all members of the server, but other factors (like command permissions) determine where and when specific members can interact with the app.

During installation, server-installed apps are authorized with a specific set of OAuth2 scopes and bot user permissions that determine what resources and data the app can access in that server.

### User context

Apps installed in a user context (user-installed apps) are visible only to the authorizing user, and therefore don't require any server-specific permissions.

Apps that support the user installation context are visible across all of an authorizing user's servers, DMs, and GDMs, but are forced to respect the user's permissions in the surface where the app is being used.

### Setting supported installation contexts

By default, newly-created apps only support installation to guilds.

You can update which installation contexts your app supports in your app's settings. On the **Installation** page under the **Installation Contexts** section, you can select the installation contexts your app supports.

<Info>
If you update your app to support a new installation context, you will need to update your existing commands if you want them to be supported in the new context.
</Info>

## Install links

Install links provide an easy way for users to install your app in Discord. If you have an install link configured, an "Add App" button will appear in your app's profile and App Directory page.

There are three options when configuring an install link: "Discord Provided Link", "Custom URL", and "None".

### Discord provided link

The default Discord Provided Link is a short link that guides users through the installation flow with your app's configured installation contexts. Discord Provided Links don't have scopes or bot user permissions defined in the URL. Instead, these links prompt the user for the scopes and bot user permissions configured in your Default Install Settings.

<Info>
Discord Provided Links are limited to the `application.commands` and `bot` scopes.
</Info>

### Custom URL

A Custom URL is an alternative to the Discord Provided Link that gives you more control of where users are directed when they click "Add App". A Custom URL is commonly an OAuth2 `/authorize` URL that has defined scopes, permissions, and an installation context (`integration_type`).

## Get current application

<Note>
**GET** `/applications/@me`
</Note>

Returns the [application](/developers/resources/application#application-object) object associated with the requesting bot user.

## Edit current application

<Note>
**PATCH** `/applications/@me`
</Note>

Edit properties of the app associated with the requesting bot user. Only properties that are passed will be updated. Returns the updated [application](/developers/resources/application#application-object) object on success.

<Note>
All parameters to this endpoint are optional.
</Note>

### JSON params

<ParamField body="custom_install_url" type="string">
  Default custom authorization URL for the app, if enabled
</ParamField>

<ParamField body="description" type="string">
  Description of the app
</ParamField>

<ParamField body="role_connections_verification_url" type="string">
  Role connection verification URL for the app
</ParamField>

<ParamField body="install_params" type="install params object">
  Settings for the app's default in-app authorization link, if enabled
</ParamField>

<ParamField body="integration_types_config" type="dictionary">
  Default scopes and permissions for each supported installation context
</ParamField>

<ParamField body="flags" type="integer">
  App's public flags. Only limited intent flags (`GATEWAY_PRESENCE_LIMITED`, `GATEWAY_GUILD_MEMBERS_LIMITED`, and `GATEWAY_MESSAGE_CONTENT_LIMITED`) can be updated via the API.
</ParamField>

<ParamField body="icon" type="?image data">
  Icon for the app
</ParamField>

<ParamField body="cover_image" type="?image data">
  Default rich presence invite cover image for the app
</ParamField>

<ParamField body="interactions_endpoint_url" type="string">
  Interactions endpoint URL for the app. Must be valid according to the Receiving an Interaction documentation.
</ParamField>

<ParamField body="tags" type="array of strings">
  List of tags describing the content and functionality of the app (max of 20 characters per tag). Max of 5 tags.
</ParamField>

<ParamField body="event_webhooks_url" type="string">
  Event webhooks URL for the app to receive webhook events
</ParamField>

<ParamField body="event_webhooks_status" type="application event webhook status">
  If webhook events are enabled for the app. `1` to disable, and `2` to enable
</ParamField>

<ParamField body="event_webhooks_types" type="array of strings">
  List of webhook event types to subscribe to
</ParamField>

## Get application activity instance

<Note>
**GET** `/applications/{application.id}/activity-instances/{instance_id}`
</Note>

Returns a serialized activity instance, if it exists. Useful for preventing unwanted activity sessions.

### Activity instance object

<ResponseField name="application_id" type="snowflake" required>
  Application ID
</ResponseField>

<ResponseField name="instance_id" type="string" required>
  Activity instance ID
</ResponseField>

<ResponseField name="launch_id" type="snowflake" required>
  Unique identifier for the launch
</ResponseField>

<ResponseField name="location" type="Activity Location object" required>
  Location the instance is running in
</ResponseField>

<ResponseField name="users" type="array of snowflakes" required>
  IDs of the users currently connected to the instance
</ResponseField>

### Activity location object

<ResponseField name="id" type="string" required>
  Unique identifier for the location
</ResponseField>

<ResponseField name="kind" type="Activity Location Kind Enum" required>
  Enum describing kind of location
</ResponseField>

<ResponseField name="channel_id" type="snowflake" required>
  ID of the channel
</ResponseField>

<ResponseField name="guild_id" type="?snowflake">
  ID of the guild
</ResponseField>

### Activity location kind enum

| Enum | Description                                        |
|------|----------------------------------------------------|
| `gc` | Location is a Guild Channel                        |
| `pc` | Location is a Private Channel, such as a DM or GDM |
